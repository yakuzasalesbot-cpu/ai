# PROJECT_CONTEXT.md

## 1. Назва проєкту

ЯКУДЗА — звірка дорожніх листів і генераторів автотранспортної роти

## 2. Що це за проєкт

Коротко:
- це робоча система для ведення 10-денних карток техніки і генераторів;
- основне джерело істини — Google-таблиця;
- результат — коректні вкладки `Реєстр`, `Залишки`, `Донесення` і фінальний Excel для друку без ручного дублювання цифр.

## 3. Поточний стан

- базова структура в Google Sheets уже існує;
- робочі вкладки техніки та генератора використовуються як первинний ввід;
- фінальні вкладки `Реєстр`, `Залишки`, `Донесення` вже є в таблиці;
- верхні поля `Дата документа` і `№ документа` для активних блоків були дозаповнені;
- частина переносів залишку між блоками вже виправлена у живій Google-таблиці;
- головна задача v1 зараз: довести таблицю до єдиного шаблону і стабільної логіки, а не ускладнювати систему новою архітектурою;
- локальний експорт/генератор існує, але його можна вважати допоміжним шаром, не головним.

## 4. Стек / інструменти

- Frontend: не застосовується
- Backend: Apps Script / локальні PowerShell-скрипти
- Bot: не застосовується
- Hosting: Google Workspace / локальна Windows-машина
- Database: Google Sheets як робоча база
- Payments: не застосовується
- CRM: не застосовується
- AI/API: ChatGPT, Codex, Claude, Perplexity, Gemini, Copilot, Claude for Excel за правилами YAKUZA AI Operating System

Офіційно дозволені AI-інструменти проєкту:
- ChatGPT
- Codex
- Claude
- Perplexity
- Gemini
- Copilot
- Claude for Excel

Принцип:
- кожен інструмент має роль;
- жоден інструмент не починає великий аналіз з нуля без контексту;
- вже проговорене не повторюється без нових даних або прямої команди.

## 5. Головні правила

- не домислювати цифри;
- не виправляти облікові дані без підтвердження, якщо зміна не є однозначним переносом;
- не ламати існуючу робочу логіку без команди;
- не дублювати одну й ту саму велику задачу паралельно між кількома LLM без ролей;
- не палити ліміти на повторний сліпий аналіз;
- не відхилятись від затвердженого плану, шляху, формату або місця зберігання без окремого підтвердження;
- після кожної ітерації фіксувати короткий ОС і наступний крок;
- перед великою задачею читати `docs/01_AI_WORK_RULES.md`, `docs/02_MULTI_LLM_PROTOCOL.md`, `docs/03_API_LIMIT_GUIDELINES.md`, `docs/04_ITERATION_PROTOCOL.md`, `docs/05_HANDOFF_PROMPTS.md`.

## 6. Стиль комунікації

Коротко, прямо, без води.
Спочатку суть, потім конкретний статус.
Якщо щось неясно — позначати припущення, а не вигадувати.

## 7. Що заборонено

- видаляти або перезаписувати робочі файли без прямої команди;
- міняти логіку звітності "для краси";
- робити великі перебудови процесу без зафіксованого плану;
- запускати дорогий багаторазовий аналіз без потреби;
- змінювати друкований шаблон Excel, якщо задача стосується тільки Google-таблиці.

## 8. Поточні пріоритети

1. Уніфікувати всі робочі вкладки техніки і генератора під один жорсткий шаблон.
2. Зафіксувати ручні поля, формульні поля і контрольні поля.
3. Перевірити переходи спідометра, мотогодин і залишку пального між 3 блоками.
4. Довести до стабільності `Реєстр`, `Залишки`, `Донесення`.
5. Лише після цього повертатись до автоматичного фінального експорту Excel.

## 9. Відкриті питання

- який саме файл вважати канонічним свіжим локальним експортом для перевірки;
- чи всі активні вкладки вже приведені до єдиної візуальної схеми;
- чи лишився реальний мінус по паливу на `КОЗАК 4277` після останніх правок у таблиці;
- чи фінальний локальний генератор потрібен зараз, чи достатньо стабілізувати Google Sheets.
